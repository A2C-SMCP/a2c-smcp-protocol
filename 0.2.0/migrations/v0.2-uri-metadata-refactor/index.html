    v0.2 迁移指南：URI 元数据重构 & 协议版本握手¶

    协议版本：0.1.x → 0.2.0 目标读者：python-sdk / rust-sdk 实现工程师 发布节奏：破坏性升级，需同步发布三方（Agent / Computer / MCP Server helper）

    对从未实现 v0.1 的 SDK——按 Greenfield 实现规范阅读

    本指南默认 SDK 已实现 v0.1 完整功能并据此描述"迁移路径"。如果你所维护的 SDK 从未实现某个子系统（典型如 Finder），那么本指南中相关章节（特别是 §4.2 DPE URI 重构、§4.3 Finder 事件重命名、§5/§6 中所有 organize_finder 相关条目）应按 Greenfield 实现规范阅读：

    • 关于"旧形式"的描述可作为反面参考（避免按 v0.1 模式实现）
    • 关于"新形式"和"行为约束"是 SDK 必须实现的目标
    • "迁移步骤"中提到的"修改 X 文件"对你来说是"新建 X 文件"

    对未实现 Window URI 解析器、organize_desktop 的 SDK 同理处理。

    1. 摘要¶

    v0.2 是一次破坏性协议升级，包含三组变更：

    1. URI 元数据重构：统一 window:// 与 dpe:// 两种 scheme 的元数据承载方式，让它们都落入 MCP 生态的标准位置（Resource._meta + Resource.annotations）而非 URI query，并移除冗余的 Finder Level 0 端点
    2. 协议版本握手：引入 Socket.IO connect 阶段的版本校验机制与 MAJOR.MINOR.PATCH 语义规范（详见 versioning.md）
    3. MCP 上游授权错误码：新增 4006 / 4007，约定授权失败的结构化反馈

    核心改动一句话版¶

    1. Window/DPE URI 变成纯标识符（不再携带元数据 query）
    2. 文档/窗口元数据下沉到 MCP Resource.annotations（MCP 标准字段优先：priority / audience / lastModified）+ Resource._meta（A2C 扩展字段：fullscreen / keywords / file_type / page_count / file_uri）
    3. Finder Level 0 移除（目录发现走 MCP resources/list，不再有 dpe://host 形式）
    4. ListFinderReq.keywords 参数重命名为 tags
    5. 新增 Socket.IO 连接必带 URL query ?a2c_version=0.2.0；Server 在 HTTP 中间件层校验，不兼容时返回 HTTP 400 + 4008
    6. 新增错误码 4006 Tool Authorization Required / 4007 Tool Authorization Failed / 4008 Protocol Version Mismatch
    7. 元数据字段分工原则：语义与 MCP 标准 annotations 对齐的字段（priority / audience / lastModified）必须放入 annotations；A2C 自定义字段（fullscreen / DPE 业务字段）放入 _meta。Window audience=["assistant"]、DPE audience=["user"]
    8. Finder 事件重命名：client:get_finder → client:list_finder；对应类型 GetFinderReq/Ret → ListFinderReq/Ret、常量 GET_FINDER_EVENT → LIST_FINDER_EVENT、Agent SDK 方法 agent.get_finder() → agent.list_finder()（语义不变，对齐"list = 集合查询 + 分页"的 REST 命名约定；get_desktop 保持不变，因为 Desktop 返回的是单一复合对象）

    2. 受影响范围¶

    2.1 代码影响矩阵¶

    Python SDK 路径基于 python-sdk 仓库实际现状；Rust SDK 列仅作示意——rust-sdk 实际目录结构由 rust-sdk 实现方维护，本文档不锚定具体路径，仅描述需要修改/新建的职能模块。

    职能模块 Python SDK 路径 Rust SDK 说明
    WindowURI 解析器 a2c_smcp/computer/utils/window_uri.py rust-sdk 决定 移除 priority / fullscreen query 解析
    DPE URI 解析器 a2c_smcp/computer/utils/dpe_uri.py rust-sdk 决定（v0.2 待新建） 拒绝 dpe://host 形式（Level 0）
    organize_desktop a2c_smcp/computer/desktop/organize.py rust-sdk 决定 从 Resource.annotations.priority 读优先级（float [0,1]）；Resource._meta.fullscreen 读全屏标记
    organize_finder a2c_smcp/computer/finder/organize.py rust-sdk 决定（v0.2 待新建） 输入改为 list[Resource]，合成 DPEDocumentSummary
    MCP Manager a2c_smcp/computer/mcp_clients/manager.py rust-sdk 决定 新增 host 唯一性校验与拒绝注册逻辑；Finder 不再读 Level 0
    Socket.IO Client a2c_smcp/computer/socketio/client.py rust-sdk 决定 on_list_finder 改用新 tags 字段

    错误码枚举 a2c_smcp/smcp.py::ErrorCode rust-sdk 决定 新增 4006 / 4007 / 4008
    协议版本常量 a2c_smcp/__init__.py::PROTOCOL_VERSION rust-sdk 决定 新增导出；SDK 包版本 MAJOR.MINOR MUST 锁定协议 MAJOR.MINOR（见 versioning.md SDK 仓库）
    Agent/Computer 连接逻辑 a2c_smcp/agent/socketio/client.py、a2c_smcp/computer/socketio/client.py rust-sdk 决定 连接时 URL 拼 ?a2c_version=...，auth 默认仅传 role（业务字段由用户回调注入，详见 data-structures.md ConnectAuth）；解析 HTTP 400 body 识别 4008 → 主动 disconnect() → 抛 ProtocolVersionError
    Server 连接处理 a2c_smcp/server/ 的 HTTP 中间件 + connect handler rust-sdk 决定（socketioxide 默认 HTTP 路径 /socket.io/） 在 Socket.IO handler 之前于 HTTP 层校验 a2c_version；connect handler 处理 auth.role；server:list_room 返回含版本
    测试 tests/unit_tests/computer/、tests/integration_tests/ rust-sdk 决定 所有涉及 URI query 解析、Level 0、keywords 参数的测试要重写；新增版本握手矩阵测试

    2.2 非代码影响¶

    • MCP Server 示例仓库（若存在）需要更新
    • 文档站点：docs/specification/、docs/guides/、docs/appendix/faq.md 已更新
    • 示例 / quickstart：需要按新模式改写

    2.3 不变的部分（明确告知，避免过度改动）¶

    • 事件名：client:get_desktop / server:update_* / notify:update_* 保持不变（client:get_finder 已重命名为 client:list_finder，见 B8）
    • Agent-facing 数据结构：DPEDocumentSummary、DPEPageSummary、DPEElementDetail、GetDeskTopRet 字段形态保持不变；ListFinderRet（原 GetFinderRet）字段形态保持不变，仅 Req/Ret 类型名随事件重命名而更改，ListFinderReq.keywords 字段改名为 tags
    • DPE Level 1/2/3 的 resources/read 响应 JSON 形态：保持不变（Agent 看到的 body 结构不变）
    • Room 模型 / Server 路由 / 更新通知语义：完全不变
    • 安全模型（零凭证传播、房间隔离）：完全不变

    3. 破坏性变更详表¶

    # 变更点 旧形式 新形式 兼容策略
    B1 Window URI query 移除 window://host/path?priority=80&fullscreen=true window://host/path 硬切换；旧 URI 解析时应丢弃 query 并记录警告
    B2 Window 元数据位置 URI query Resource.annotations.priority（float [0,1]） + Resource._meta.fullscreen + Resource.annotations.audience=["assistant"] 硬切换；priority 语义由 int [0,100] 迁移到 MCP 标准 float [0,1]（值越大越优先），audience 固定 ["assistant"]（Window 面向 Agent 渲染）
    B3 DPE Level 0 移除 dpe://host → 文档目录 JSON 不存在；文档发现走 resources/list + Resource._meta/annotations 硬切换；MCP Server 收到 dpe://host 返回 -32002 Resource not found
    B4 DPE 文档元数据位置 Level 0/1 response body 中的扁平字段 Resource._meta.{keywords, file_type, page_count, file_uri} + Resource.annotations.lastModified + Resource.annotations.audience=["user"] 硬切换；Level 1 body 仍可保留扁平字段作为 Agent-facing 视图，但真值以 Resource 元数据为准；audience 固定 ["user"]（DPE 文档是用户数据资产）
    B5 ListFinderReq 字段名 keywords: list[str] tags: list[str] 硬切换；语义不变
    B6 host 唯一性 SHOULD（v0.1 后期加） MUST 全局唯一；冲突时 Computer MUST 拒绝后注册者 新增约束，旧实现若无冲突场景不受影响
    B7 Socket.IO 连接握手 无版本校验，任意客户端可连接 必须在 URL query 携带 ?a2c_version=...；Server HTTP 中间件层校验，不兼容返回 HTTP 400 + 4008 硬切换；旧客户端未带 a2c_version 被拒（HTTP 400）
    B8 Finder 事件重命名 client:get_finder / GetFinderReq / GetFinderRet / GET_FINDER_EVENT / agent.get_finder() client:list_finder / ListFinderReq / ListFinderRet / LIST_FINDER_EVENT / agent.list_finder() 硬切换；语义不变。get_desktop 保持不变（因返回单一复合对象而非集合）
    N1 MCP 上游授权错误码 泛化为 4003 4006 / 4007（独立码） 新增；旧 Computer 返回 4003 在 Agent 侧仍可工作但体验降级
    N2 SessionInfo 字段扩展 {sid, name, role, office_id} 新增 a2c_version 新增字段，对老客户端向后兼容（可忽略不识别字段）

    N1 / N2 是新增能力而非破坏性变更；单独列出以便整理。

    4. 详细变更规范¶

    4.1 Window URI 重构¶

    URI 语法¶

    // 旧
window://host/path1/path2?priority=P&fullscreen=F

// 新
window://host/path1/path2

    校验规则¶

    • 解析时若 URI 含 query 部分，MUST 丢弃并记录 WARNING 级日志
    • path 段继续支持 URL 编码（c%2Fd 保持为单段）
    • host 在同一 Computer 作用域内 MUST 全局唯一

    元数据声明（由 MCP Server 在 resources/list 响应中设置）¶

    字段 类型 必需 默认 说明
    Resource.annotations.priority float [0, 1] 否 0.0 MCP 标准；布局排序依据，值越大越优先，仅同一 Server 内比较
    Resource.annotations.audience ["assistant"] 推荐 — MCP 标准；Window 面向 Agent 渲染，MCP Server 应显式声明 ["assistant"]（表示资源给 Agent 使用）
    Resource.annotations.lastModified ISO 8601 否 — MCP 标准；v1 Computer 透传不排序
    Resource._meta.fullscreen bool 否 false A2C 扩展；全屏渲染标记，无对应 MCP 标准字段故保留在 _meta

    priority 语义变化：v0.1 在 URI query 中以 int [0, 100] 表达，v0.2 迁移到 MCP 标准 annotations.priority（float [0, 1]，值越大越优先）。两者语义一致（数值越大优先级越高），但值域不同，MCP Server 升级时需要做值域转换（例如 new = old / 100）。

    organize_desktop 行为变更¶

    • 输入：windows: list[tuple[SERVER_NAME, Resource, ReadResourceResult]]（Resource 对象必须完整携带 annotations 与 _meta）
    • 步骤 3 改为：sorted(windows, key=lambda w: (w.resource.annotations.priority if w.resource.annotations else None) or 0.0, reverse=True)
    • 步骤 4 改为：检查 w.resource._meta.get("fullscreen", False)

    不再解析 URI query。

    4.2 DPE URI 重构¶

    URI 语法¶

    // 旧（四级）
dpe://host                              → Level 0 (目录)
dpe://host/doc-ref                      → Level 1
dpe://host/doc-ref/pages/N              → Level 2
dpe://host/doc-ref/elements/ID          → Level 3

// 新（三级，Level 0 移除）
dpe://host/doc-ref                      → Level 1
dpe://host/doc-ref/pages/N              → Level 2
dpe://host/doc-ref/elements/ID          → Level 3

    解析器行为¶

    • 当 URI 缺少 doc-ref（即 dpe://host 形式）：MUST 返回解析失败
    • Content-control query 参数（format / depth / offset / limit / categories）继续支持
    • host 唯一性 MUST 约束同 Window

    MCP Server resources/read 实现变更¶

    # 旧分支
if parsed.doc_ref is None:
    # Level 0: 返回整个文档目录
    return ReadResourceResult(contents=[TextResourceContents(
        text=json.dumps(get_document_catalog()),
        ...
    )])

# 新分支
if parsed.doc_ref is None:
    # Level 0 已移除
    raise ResourceNotFoundError(uri)  # MCP -32002

    文档元数据声明（替代旧的 Level 0 body）¶

    每个文档作为一个 Resource 出现在 resources/list 响应中：

    Resource(
    uri="dpe://com.example.docs/rpt-2026",
    name="2026 年度报告",                              # = DPE title
    description="2026 年度财务与运营报告",              # = DPE summary
    mimeType="application/json",
    annotations=Annotations(
        audience=["user"],                            # DPE 文档是用户数据资产，固定 ["user"]
        lastModified="2026-01-15T08:30:00Z",          # Finder 排序依据
    ),
    _meta={
        "keywords": ["财务", "年报"],                  # Finder tags 过滤对象之一
        "file_type": "xlsx",                          # Finder file_type 过滤对象
        "page_count": 12,                             # 必需：Agent 规划翻页
        "file_uri": "file:///data/reports/2026-annual.xlsx",  # 可选
    },
)

    DPEDocumentSummary 合成规则（Computer 实现）¶

    Computer 从 Resource 合成 DPEDocumentSummary：

    DPEDocumentSummary 字段 来源
    doc_ref 从 Resource.uri 解析得出
    uri Resource.uri
    file_uri Resource._meta.get("file_uri")
    file_type Resource._meta.get("file_type")
    title Resource.name
    page_count Resource._meta["page_count"]（必需；缺失则 Resource 被 Organizer 过滤掉）
    keywords Resource._meta.get("keywords")
    summary Resource.description
    server Computer 注入（来源 MCP Server 名称）
    last_modified Resource.annotations.get("lastModified")

    4.4 MCP Server host 唯一性约束¶

    约束¶

    在同一 Computer 范围内：

    • 所有 window:// URI 的 host 全局唯一
    • 所有 dpe:// URI 的 host 全局唯一
    • 跨 scheme 不冲突（同一 host 可以同时有 window:// 和 dpe://）

    Computer 行为¶

    在 MCP Server 注册流程中，Computer 必须：

    1. 列举该 Server 将要声明的所有 window:// 和 dpe:// 资源的 host 集合
    2. 与已注册 Server 的 host 集合求交集
    3. 有交集：
    4. 记录 ERROR 级日志（包含冲突 host 列表、已占用的 Server 名、新 Server 名）
    5. 拒绝该 Server 参与 Desktop / Finder 聚合（仅保留已注册的一方）
    6. 其他 scheme（如工具调用）不受影响

    实现建议¶

    • 在 MCPServerManager.register_server 或等价入口加唯一性检查
    • 维护一张 host → (scheme, server_name) 索引
    • Server 下线时释放索引

    4.5 错误码新增（MCP 上游授权）¶

    码表¶

    码 名称 语义
    4006 Tool Authorization Required MCP 上游需授权（如 OAuth 2.0），Computer 当前无有效凭证或未完成授权
    4007 Tool Authorization Failed OAuth 流程失败、Token 失效或刷新失败

    CallToolResult 响应结构¶

    CallToolResult(
    content=[TextContent(text="此工具需要授权 GitHub 账号后方可使用")],
    isError=True,
    meta={
        "error_code": 4006,  # 或 4007
        "mcp_server": "github-mcp",
        "auth_hint": {
            "action": "user_authorization_required",
            "message": "请在 Computer 宿主环境完成 GitHub OAuth 登录后重试"
        }
    }
)

    字段约束¶

    • meta.error_code：必需，整数 4006 或 4007
    • meta.mcp_server：必需，字符串
    • meta.auth_hint：可选，对象；禁止包含授权 URL 参数、state、client_secret、code_verifier、access_token 等敏感字段

    Computer 实现要求¶

    • MCP Server SDK 调用 Tool 时捕获上游授权异常（httpx.HTTPStatusError(401/403) 等）
    • 分类判定：未登录 → 4006；Token 失效或刷新失败 → 4007
    • 禁止泛化为 4003（Tool Execution Failed）
    • OAuth 握手流程：必须在 Computer 宿主环境完成，凭证不经 A2C-SMCP 信道传输

    4.6 协议版本握手（新增）¶

    完整规范见 协议版本与握手。本节摘录 SDK 需关注的实现要点。

    版本号语义¶

    • 格式：MAJOR.MINOR.PATCH
    • v0.x 阶段（当前）：MAJOR.MINOR 必须严格匹配（如 0.2.0 仅兼容 0.2.x），PATCH 可自由差异
    • v1.0+（未来）：MAJOR 严格匹配；同 MAJOR 内 Server MINOR ≥ Client MINOR

    关键设计点¶

    1. 版本号放在 URL query，不在 auth 对象里
    2. URL query 在 HTTP 层，先于 Socket.IO 业务代码执行
    3. auth 对象是业务层 envelope，用于 role 等身份数据

    4. 这样保证版本校验不受业务代码 bug 影响

    5. Server 侧校验在 HTTP 中间件层，不在 connect handler

    6. 中间件在 Socket.IO ASGI/WSGI app 外层挂载
    7. 版本不兼容的请求在 HTTP 层直接被拒（HTTP 400），不进入 Socket.IO

    Client 侧 SDK 实现¶

    1. 导出 PROTOCOL_VERSION 常量（Python 包级 / Rust crate 级）
    2. 连接时自动在 URL query 拼接版本：

    f"wss://server.example.com?a2c_version={PROTOCOL_VERSION}",

    1. 识别 connect_error 的 HTTP 400 payload：
    import json

class ProtocolVersionError(ConnectionError):
    def __init__(self, server_version, client_version, message):
        self.server_version = server_version
        self.client_version = client_version
        super().__init__(message)

@sio.on("connect_error")
async def on_connect_error(data):
    payload = data if isinstance(data, dict) else (
        json.loads(data) if isinstance(data, str) else None
    )
    if payload and payload.get("code") == 4008:
        raise ProtocolVersionError(
            server_version=payload.get("server_version"),
            client_version=payload.get("client_version"),
            message=payload.get("message", "Protocol version mismatch"),
        )
    1. 禁止在收到 4008 时自动重试。SDK 处理流程：解析 HTTP 400 body → 识别 code:4008 → 主动调用底层客户端 disconnect()（避免库默认重连导致死循环）→ 抛 ProtocolVersionError 给业务层。详见 versioning.md § Client SDK MUST 主动断开连接

    Server 侧 SDK 实现¶

    from starlette.middleware import Middleware
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.responses import JSONResponse

# Python 实现选择：把 socketio_path 设为 "smcp"，故 HTTP 路径前缀也是 /smcp
# 其他 SDK（Rust socketioxide / Node socket.io）通常用 /socket.io/，对应调整下方过滤
SMCP_HTTP_PATH = "/smcp"

class A2CVersionMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        if not request.url.path.startswith(SMCP_HTTP_PATH):
            return await call_next(request)

        client_ver_str = request.query_params.get("a2c_version")
        if not client_ver_str:
            return JSONResponse(
                {"code": 400, "message": "Missing a2c_version query parameter"},
                status_code=400,
            )
        try:
            client_ver = ProtocolVersion.parse(client_ver_str)
        except ValueError as e:
            return JSONResponse(
                {"code": 400, "message": f"Invalid a2c_version: {e}"},
                status_code=400,
            )
        if not is_compatible(client_ver, SERVER_VERSION):
            return JSONResponse(
                {
                    "code": 4008,
                    "message": "Protocol version mismatch",
                    "server_version": str(SERVER_VERSION),
                    "client_version": str(client_ver),
                },
                status_code=400,
            )
        return await call_next(request)

# ASGI 组装
app = Starlette(
    middleware=[Middleware(A2CVersionMiddleware)],
    routes=[Mount("/", socketio.ASGIApp(sio, socketio_path="smcp"))],
)

    兼容性判定函数（共享代码参考）¶

    @dataclass(frozen=True)
class ProtocolVersion:
    major: int
    minor: int
    patch: int

    @classmethod
    def parse(cls, s: str) -> "ProtocolVersion":
        parts = s.split(".")
        if len(parts) != 3:
            raise ValueError(f"Invalid version: {s}")
        return cls(int(parts[0]), int(parts[1]), int(parts[2]))

    def __str__(self) -> str:
        return f"{self.major}.{self.minor}.{self.patch}"


def is_compatible(client: ProtocolVersion, server: ProtocolVersion) -> bool:
    if client.major != server.major:
        return False
    if client.major == 0:
        return client.minor == server.minor  # v0.x 严格
    return client.minor <= server.minor  # v1.0+ 向后兼容

    7. 兼容性策略¶

    7.1 协议层：硬切换¶

    v0.1.x 与 v0.2.0 在 URI、ListFinderReq 字段等层面无法互通。SDK 行为：

    1. HTTP 层版本握手：客户端在 Socket.IO 连接 URL query 中携带 ?a2c_version=...；Server 中间件层校验，不兼容时返回 HTTP 400 + 错误码 4008 Protocol Version Mismatch（详见 versioning.md）。这是 v0.2 引入的标准机制
    2. server:list_room 返回的 SessionInfo 含 a2c_version 字段，便于 Agent/Computer 在房间内做二次确认（兜底诊断用）

    不内建 v0.1 MCP Server fallback（协议级决议）¶

    v0.2 SDK MUST NOT 内建针对 v0.1 MCP Server 的 URI query fallback 解析。即：当 v0.2 Computer 遇到只在 URI query 中携带 ?priority= / ?fullscreen= 的 v0.1 MCP Server 时，MUST：

    • 按 v0.2 校验规则丢弃 URI query 部分（仅记录 WARN）
    • 不读取 _meta / annotations 之外的元数据来源
    • Desktop 中该窗口降级显示为 priority=0.0, fullscreen=false

    理由（保持协议纯粹）：

    • 内建 fallback 会让"v0.2 协议是什么"在不同 SDK 实现间产生歧义（实现 A 兼容 v0.1 query、实现 B 不兼容）
    • 用户感知到降级行为后会主动升级 MCP Server 到 v0.2，比静默兼容更早暴露问题
    • v0.2 协议升级周期短，长期维护两套元数据通路收益低、复杂度高

    SDK 实现方若有强烈业务诉求，可在 SDK 之外的应用层提供"v0.1 MCP Server 适配器"作为可选扩展，不要污染协议核心代码。

    7.2 数据层：Agent 视角兼容¶

    • DPEDocumentSummary 字段形态不变，Agent 端消费代码无需改造
    • 仅 ListFinderReq.keywords → tags 是 Agent 端必须的改动
    • 4006/4007 是新增码，旧 Agent 遇到会降级为"未知错误"，功能上不至于崩溃

    7.3 MCP Server 生态兼容¶

    • 旧 MCP Server（只输出 URI query 形态）在 v0.2 Computer 下 Desktop MUST 降级为 priority=0.0, fullscreen=false 并记录 WARN（协议禁止 fallback，见 §7.1），不会崩溃；用户因此感知到降级会主动升级 MCP Server
    • 旧 MCP Server（实现 Level 0）在 v0.2 Computer 下，Level 0 响应会被忽略（Computer 不再调用），不会报错

    结论：v0.2 Computer 对旧 MCP Server 宽容（降级运行），但 v0.1 Computer 无法消费 v0.2 MCP Server（因元数据在 URI 之外，旧 Computer 看不到）。因此升级顺序是 Computer 先于 MCP Server。

    7.4 建议的升级顺序¶

    阶段 1：发布 v0.2 Computer SDK（支持新模型 + 对旧 MCP Server 降级）
    ↓
阶段 2：用户升级 Computer 到 v0.2
    ↓
阶段 3：发布 v0.2 MCP Server Helper
    ↓
阶段 4：MCP Server 逐步升级到 v0.2
    ↓
阶段 5：发布 v0.2 Agent SDK（ListFinderReq.tags）
    ↓
阶段 6：Agent 升级到 v0.2

    async def organize_desktop(
    windows: list[tuple[str, Resource, ReadResourceResult]],
    size: int | None,
    history: tuple[ToolCallRecord, ...],
) -> list[str]:
    # ...
    for server_name, resource, details in windows:
        parsed = WindowURI.parse(str(resource.uri))  # 解析器不再提取元数据
        # priority 从 MCP 标准 annotations 读取（float [0, 1]）
        ann = resource.annotations
        priority = getattr(ann, "priority", None) if ann else None
        if priority is None:
            priority = 0.0
        elif not isinstance(priority, (int, float)) or not (0.0 <= float(priority) <= 1.0):
            logger.warning("Invalid annotations.priority on %s", resource.uri)
            priority = 0.0
        else:
            priority = float(priority)
        # fullscreen 是 A2C 扩展，留在 _meta
        meta = resource.meta or {}
        fullscreen = bool(meta.get("fullscreen", False))
        # ...

# 识别 HTTP 400 + 4008
@sio.on("connect_error")
async def on_connect_error(data):
    payload = data if isinstance(data, dict) else (
        json.loads(data) if isinstance(data, str) else None
    )
    if payload and payload.get("code") == 4008:
        raise ProtocolVersionError(
            server_version=payload.get("server_version"),
            client_version=payload.get("client_version"),
        )
